# scir-signal-build

Build-time helper for emitting Rust source files containing SciPy-parity biquad coefficient tables.

Designed for the **static-overlay** pattern in embedded / `no_std` DSP runtimes that cannot link `scir-signal` (which transitively depends on `ndarray`) but still want SciPy-validated coefficients pinned to a known scir version.

## Usage

In your consumer crate's `Cargo.toml`:

```toml
[build-dependencies]
scir-signal-build = "0.0.1"
```

In `build.rs`:

```rust
use std::path::PathBuf;
use scir_signal_build::{Emitter, FilterType};

fn main() -> std::io::Result<()> {
    let out = PathBuf::from(std::env::var_os("OUT_DIR").unwrap()).join("baked_filters.rs");
    Emitter::new()
        .header("Generated for the audio anti-alias chain at 48 kHz.")
        .add_butter("ANTI_ALIAS_48K_BUTTER4", 4, FilterType::LowPass(1_000.0 / 24_000.0), 2.0)
        .unwrap()
        .add_cheby1("EQ_HPF_CHEBY1_4", 4, 1.0, FilterType::HighPass(80.0 / 24_000.0), 2.0)
        .unwrap()
        .write(&out)?;
    Ok(())
}
```

In `lib.rs`:

```rust
include!(concat!(env!("OUT_DIR"), "/baked_filters.rs"));

// ANTI_ALIAS_48K_BUTTER4 is now `pub static [...]: [[f64; 6]; 2]`.
```

## Output format

Every emitter call produces a Rust source file with a stable, deterministic shape:

```rust
// Generated by scir-signal-build@0.0.1 — do not edit by hand.
// Generation is deterministic; same inputs always produce identical output.
//
// Generated for the audio anti-alias chain at 48 kHz.

/// butter(order=4, kind=LowPass(0.04167), fs=2.0)
pub static ANTI_ALIAS_48K_BUTTER4: [[f64; 6]; 2] = [
    [9.46233324867344e-6, 1.89246664973469e-5, ..., -1.93393e0, 9.36042e-1],
    [...],
];
```

Coefficients use `{:.17e}` (17-significant-digit scientific notation), which losslessly round-trips f64 IEEE values. Output is byte-identical for byte-identical inputs, so the file MAY be checked into git as a drift-test artifact.

## When to use this vs `scir-signal` directly

- **Use `scir-signal-build`** if your runtime is `no_std`, builds for an MCU/DSP/FPGA target, or you want compile-time-pinned coefficients with no runtime design code.
- **Use `scir-signal` directly** if your runtime can link `ndarray` and you need to design coefficients dynamically (e.g. cutoff varies at runtime).

## Relationship to scir issue #03 (Option B)

This crate is the **Option B** half of [scir-issues/03](https://github.com/SoftOboros/scir/issues) — "build-script helper crate". The principled long-term answer (**Option A**, a no_std-compatible `scir-signal-design` crate) remains future work; the bottleneck there is `scir-iir-filters` itself depending on `std::f64` transcendental methods (sinh, cosh, asinh, etc.) and `String`/`format!` for error messages, which would need a libm-backed conversion pass.

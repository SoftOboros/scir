# scir-signal-build

Build-time helper for emitting Rust source files containing SciPy-parity biquad coefficient tables, STFT window amplitude tables, and FFT bin label tables.

Designed for the **static-overlay** pattern in embedded / `no_std` DSP runtimes that cannot link `scir-signal` (which transitively depends on `ndarray`) but still want SciPy-validated coefficients pinned to a known scir version.

## Usage

In your consumer crate's `Cargo.toml`:

```toml
[build-dependencies]
scir-signal-build = "0.0.4"
```

In `build.rs`:

```rust
use std::path::PathBuf;
use scir_signal_build::{Emitter, BesselNorm, FftLayout, FilterType, WindowShape};

fn main() -> std::io::Result<()> {
    let out = PathBuf::from(std::env::var_os("OUT_DIR").unwrap()).join("baked.rs");
    Emitter::new()
        .header("Generated for the audio anti-alias chain at 48 kHz + STFT @ N=4096.")
        .add_butter("ANTI_ALIAS_48K_BUTTER4", 4, FilterType::LowPass(1_000.0 / 24_000.0), 2.0)
        .unwrap()
        .add_cheby1("EQ_HPF_CHEBY1_4", 4, 1.0, FilterType::HighPass(80.0 / 24_000.0), 2.0)
        .unwrap()
        .add_bessel_with_norm(
            "BESSEL3_BPF",
            3,
            BesselNorm::Delay,
            FilterType::BandPass(1_500.0 / 24_000.0, 5_000.0 / 24_000.0),
            24_000.0,
        )
        .unwrap()
        .add_iirnotch("Mains50", 50.0, 30.0, 24_000.0)
        .unwrap()
        .add_window("HANN_4096", WindowShape::Hann, 4096)
        .as_f32()
        .add_window("BH4_4096", WindowShape::BlackmanHarris4Term, 4096)
        .as_f32()
        .add_fft_bin_labels("BIN_HZ_4096", 4096, 48_000.0, FftLayout::OneSidedReal)
        .as_f32()
        .write(&out)?;
    Ok(())
}
```

In `lib.rs`:

```rust
include!(concat!(env!("OUT_DIR"), "/baked.rs"));

// ANTI_ALIAS_48K_BUTTER4 is now `pub static [...]: [[f64; 6]; 2]`.
// HANN_4096                is now `pub static [...]: [f32; 4096]`.
// BIN_HZ_4096              is now `pub static [...]: [f32; 2049]` (N/2+1 bins, DC..Nyquist).
```

## Output format

Every emitter call produces a Rust source file with a stable, deterministic shape:

- Biquad SOS items emit nested `[[T; 6]; N]`.
- Window and bin-label items emit flat `[T; N]`.

`Emitter` now includes build-time parity for the new runtime APIs:

- `add_bessel_with_norm(ident, order, norm, kind, fs)` for explicit Bessel normalization.
- `add_iirnotch(ident, w0, q, fs)` for narrow notch filters.

In this form:

```rust
// Generated by scir-signal-build@0.0.4 — do not edit by hand.
// Generation is deterministic; same inputs always produce identical output.

/// butter(order=4, kind=LowPass(0.04167), fs=2.0)
pub static ANTI_ALIAS_48K_BUTTER4: [[f64; 6]; 2] = [
    [9.46233324867344e-6, 1.89246664973469e-5, ..., -1.93393e0, 9.36042e-1],
    [...],
];

/// hann(length=4096)
/// (truncated from f64 to f32 at emission)
pub static HANN_4096: [f32; 4096] = [
    0.000000000e0_f32,
    1.471878760e-7_f32,
    /* ... */
];
```

Coefficients use `{:.17e}` (17-significant-digit scientific notation) for f64, which losslessly round-trips IEEE values. Output is byte-identical for byte-identical inputs, so the file MAY be checked into git as a drift-test artifact.

## Window functions

Three window shapes ship in v0.0.4+ (frozen by ASA-01 spec; see consumer-side spec lineage):

- `WindowShape::Hann` — generic-purpose STFT.
- `WindowShape::BlackmanHarris4Term` — high dynamic range (~92 dB side-lobe rejection).
- `WindowShape::FlatTop` — amplitude-accurate single-tone measurement.

Each is closed-form per `scir_signal::window::window`. `Emitter::add_window` evaluates the closed form at the consumer's build time and emits a single-array static table the runtime consumes by index.

## FFT bin labels

`Emitter::add_fft_bin_labels(ident, n, fs, layout)` (v0.0.4+, ASA-02 spec) emits a `pub static IDENT: [f32; M]` (or f64) of bin center frequencies in Hz. For `FftLayout::OneSidedReal` (the v1 default and only variant), `M = N/2 + 1` covering DC through Nyquist inclusive. Spectrum-display widgets index this array in parallel with the FFT engine's magnitude output.

## Release notes

`v0.3.5` includes all previous 0.0.x behavior plus:

- `add_bessel_with_norm` for explicit Bessel `Phase | Delay | Mag` parity.
- `add_iirnotch` for SciPy-style notch designs.
- Header docs and output examples updated for `v0.0.4` codegen versioning.

## When to use this vs `scir-signal` directly

- **Use `scir-signal-build`** if your runtime is `no_std`, builds for an MCU/DSP/FPGA target, or you want compile-time-pinned coefficients with no runtime design code.
- **Use `scir-signal` directly** if your runtime can link `ndarray` and you need to design coefficients dynamically (e.g. cutoff varies at runtime).

## Relationship to scir issue #03 (Option B)

This crate is the **Option B** half of [scir-issues/03](https://github.com/SoftOboros/scir/issues) — "build-script helper crate". The principled long-term answer (**Option A**, a no_std-compatible `scir-signal-design` crate) remains future work; the bottleneck there is `scir-iir-filters` itself depending on `std::f64` transcendental methods (sinh, cosh, asinh, etc.) and `String`/`format!` for error messages, which would need a libm-backed conversion pass.
